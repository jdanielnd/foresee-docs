To generate forecasts, you must first load the historical data of the series that you wish to forecast. This loading is made using a CSV data file.

This CSV file can easily be created from a Microsoft Excel ® spreadsheet. Once the dates and values ​​of the series are correctly entered on this worksheet, simply click (while in Excel) on File > Save as ... >, then select the CSV option on Type.

After loading the data file in Foresee, the whole process is automated, and the forecasts are generated in approximately one minute.

However, it is important to pay attention to the formatting guidelines listed below:

  1. File Structure

    1. The first line should contain the header with the title of the columns arranged below it.
    2. The first column should contain the dates of series.
    3. The following columns (column 2 onwards) should contain the values ​​of the series (see illustration below).

  2. Formatting of first line or header

    The formatting of column headings is quite flexible.

    You can use upper or lowercase letters, as well as numbers, spaces, periods, hyphens and subscripts.

    It is important to note that it is not possible to create two or more series with the same name within the same data file.

  3. Formatting the historical data

    The data can represent:

    1. Monetary values​​: for example, the value of the monthly sales;
    2. Quantities: monthly units sold for item "x";
    3. Percentage: monthly contribution margin.

    In all these cases the series:

    1. Must contain only numeric values​​,
    2. May contain point or comma as decimal separator,
    3. Cannot contain any thousands separator,
    4. Cannot contain any symbols such as £, US$, %, etc.

    When expressing percentages, the data must be represented as decimal numbers (for example, as 0.134 or 0.625, and not 13.4% or 62.5%).

  4. Formatting dates in the CSV data file

    Dates should always include a day, a month and a year. Foresee recognizes four different date formats.

    For example, to represent June 30, 2010, you can use any one of the following formats:

    Format Date
    DD/MM/YYYY 30/06/2010
    MM/DD/YYYY 06/30/2010
    DD/MM/YY 30/06/10
    MM/DD/YY 06/30/10

  5. Formatting dates in Foresee

    For dates reported in the data file to be interpreted correctly, it is necessary to let Foresee know the date format you'll be working with before loading the file.

    By clicking Settings > Date format (see picture below), you can select the format with which you wish to work.

    This date configuration is saved in your profile. You only need to repeat this procedure if you want to load a data file with a new date format.

  6. Series frequency

    Foresee allows you to work with monthly, quarterly, semiannual or annual data.

    To represent any of these periods (month, quarter, semester, or year), you can refer (a) to the beginning of the period or (b) to the end of the period.

    Foresee will work both ways. You can use either of these two criteria, as long as it is maintained consistently throughout the series.

    As indicated in the table above, in order to represent, for example, the months of January, February and March of 2013, you can use 01/01/2013, 02/01/2013 and 03/01/2013, referring to the beginning of these periods.

    Alternatively, you can also express these months by referring to the end of the period (i.e., 01/31/2013, 02/28/2013 and 03/31/2013).

    Both of these forms for representing time will work with Foresee. In both cases, it will understand that you are referring to the months of January, February and March, whether the reference is made to the beginning or end of the period, as long as the criterion adopted is consistent throughout the series.

    Similarly, to refer to the 1st, 2nd and 3rd quarters of 2013, you may refer to the beginning of the quarters (01/01/2013, 04/01/2013 and 07/01/2013) or if you prefer, you can refer to the end of the quarter (03/31/2013, 06/30/2013 and 09/30/2013).

    In both cases, Foresee will understand that you are referring to the 1st, 2nd and 3rd quarters of 2013.

    Tip: If you always choose to work with the starting point of the period, you won't have to ask yourself, "What was the last day of each period" (28, 29, 30 or 31?). You will always use the 1st day of the month when dealing with monthly data or the 1st day of the starting month of each period when dealing with quarterly, semi-annual, or annual data.

  7. Minimum and maximum size of the series

    1. Minimum number of periods

      To be able to forecast the values ​​of a series, Foresee needs to access the values of at least 5 consecutive previous periods from this series (five months, five quarters, five semesters or five years, for monthly, quarterly, semiannual or annual data, respectively).

      However, it is important to remember that the longer the historical data, the better the forecasts will be.

    2. Seasonality and minimum number of periods

      To be able to detect the seasonality of a series, Foresee needs to have access to at least 2 full years of information (24 months, 8 quarters, etc.).

      This does not mean that you cannot generate forecasts with less than 24 months or fewer than 8 quarters. If the series to be forecasted do not show seasonality, this will have no major effect on the quality of the forecasts.

      However, if the series you are forecasting show an important seasonal pattern, it will not be represented in the forecasts, which means larger prediction errors.

      This does not apply to annual data, since these do not exhibit seasonality.

    3. Maximum number of periods

      Foresee allows you to load data files with series composed of up to 120 periods (120 months, 120 quarters, etc..).

      However, this does not mean that you have to work with such a large number of periods in order to generate good quality forecasts.

      If your data are monthly, for example, you can generate good quality forecasts by working with a history of 48 or even 24 months.

      Nevertheless, it is important to emphasize that the quality of the forecasts will improve as you increase the number of periods.

  8. Maximum number of series in a single data file

    Foresee allows you to load data files containing up to 20 different series each time.

    The forecasts for these series are computed simultaneously, and they will be saved within the same project on Foresee.

    This does not mean that you are limited to 20 forecasts. In fact, you can generate as many forecasts as you wish, but each data file (and consequently, each project) may not contain more than 20 different series.

  9. The series that comprise a single data file must have the same frequency and must occur at the same time.

    The different series in a data file share a single column of dates. This means that these series:

    1. Should have the same frequency, be it monthly, quarterly, half-yearly or yearly.
    2. Should begin and end on the same dates.
    3. Should have the same length.

  10. "Missing" data ​​and "zero" values

    Missing values occur when no data value is available for this series in the current observation. This happens, for example, when a company does not store the number of units sold for a specific itemin a given month.

    In such case, Foresee cannot generate a forecast because there is a "missing value" or "gap" in the historical data. The series must be complete, with no "missing" values ​​or periods.

    Note that this is different from a value equal to "zero".

    This happens, for example, when a product's sale have been zero (i.e., the company did not sell any units of that product).

    This value should be reported in the data file as "0" (number zero) and not as a blank field or "missing value", which are not allowed in Foresee.
